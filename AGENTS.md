# Crescia V1— Agents.md

# Instructions for AI coding agents (Claude Code, Cursor Composer, etc.)

## WHAT THIS PROJECT IS

Crescia is a web-based exam prep platform for AP Statistics, AP Biology, and AP Calculus AB/BC.
It combines AI-generated practice questions, adaptive performance tracking, and a garden gamification
system where students grow virtual plants as they study. Built by a solo developer. MVP target: 8 weeks.

---

## HOW TO WORK ON THIS PROJECT

### Before writing any code

1. Read .cursorrules in the project root. All architecture and coding rules live there.
2. Read /types/index.ts to understand existing data shapes before defining new ones.
3. Check /lib/supabase/schema.sql to understand the database structure.
4. Never create a file without checking if one already exists for that purpose.

### When given a task

- Confirm which files you will create or modify before writing code.
- Build the minimum version that satisfies the task. Do not add unrequested features.
- After completing, list what you built, what you changed, and what the agent after you should know.

## MODULE BREAKDOWN

### 1. Authentication (Clerk)

- Clerk handles all auth. Do not touch this unless adding new webhook events.
- On user.created webhook: insert row into public.users table in Supabase.
- Middleware lives in /middleware.ts — protects all /dashboard/* routes.
- File: /app/api/webhooks/clerk/route.ts

### 2. Question Engine

- Questions are generated by Claude (claude-sonnet-4-20250514) via Server Actions.
- System prompt is cached using cache_control: ephemeral to reduce API costs.
- Generated questions are stored in questions_cache table before being served.
- On question request: check cache first, generate only on cache miss.
- Web scraping for question bank lives in /lib/scraper — runs as a build-time or scheduled job, NOT on user request.
- File: /app/api/questions/generate/route.ts, /lib/ai/question-generator.ts

### 3. FRQ Grader (AP Stats only in MVP)

- Student submits a written answer. Claude scores it against AP rubric criteria.
- Returns: { score, max_score, feedback[], rubric_points[] }
- Prompt caching applies to the rubric system prompt.
- File: /app/api/questions/grade/route.ts, /lib/ai/frq-grader.ts

### 4. Session Flow

- A session = one study block. Has a subject, mode (timed/untimed), question array, answers.
- On session complete: save to sessions table, save each answer to answers table, update topic_stats.
- Readiness score is recalculated after every session completion.
- File: /app/dashboard/study/page.tsx, /app/api/sessions/complete/route.ts

### 5. Gamification Engine

- XP: awarded only for correct answers. Harder questions = more XP. Weak topics = 1.5x bonus.
- Streaks: daily check on session completion. Grace day logic: 1 per week.
- Achievements: checked on every session completion. Unlock conditions are hardcoded in /lib/gamification/achievements.ts.
- Readiness score: weighted average across topic_stats for a subject. Stored in users table per subject.
- File: /lib/gamification/xp.ts, /lib/gamification/streaks.ts, /lib/gamification/achievements.ts

### 6. Garden System 

- Each user has a garden with plots. Each plot can hold one plant.
- Plants have 5 growth stages: Seed → Sprout → Budding → Flowering → Flourishing.
- Growth is triggered by: streak maintenance, daily challenge completion.
- Dormancy: plant goes dormant after 3 days of inactivity. Revival requires a Revival Potion item.
- Shards (currency): earned by correct answers and challenges. Spent in the Seed Shop for new plants.
- File: /app/dashboard/garden/page.tsx, /lib/gamification/garden.ts, /components/garden/*

### 7. Stripe Paywall

- Stripe is TEST MODE only. No real payments in MVP.
- On checkout: create Stripe customer, create subscription, save to subscriptions table.
- Webhook handles: checkout.session.completed, customer.subscription.deleted, customer.subscription.updated.
- Plan check: /lib/stripe/plan.ts exports getUserPlan(userId) — used in all gated Server Actions.
- File: /app/api/webhooks/stripe/route.ts, /lib/stripe/plan.ts

### 8. Analytics / Heatmap

- Topic heatmap: reads from topic_stats table. Color-coded by accuracy (red < 50%, yellow 50-75%, green > 75%).
- Readiness score history: reads from sessions table, plots score over time using Recharts.
- File: /components/analytics/TopicHeatmap.tsx, /components/analytics/ReadinessScore.tsx

## DATABASE QUICK REFERENCE


| Table           | Purpose                                                                  |
| --------------- | ------------------------------------------------------------------------ |
| users           | Core user profile: xp, level, streak, plan, readiness scores per subject |
| sessions        | Each study session: subject, mode, score, timing                         |
| answers         | Per-question answers: correct, time spent, topic tag                     |
| topic_stats     | Aggregated accuracy per user per topic per subject                       |
| questions_cache | Cached AI-generated questions by subject/topic/difficulty                |
| achievements    | Unlocked achievement records per user                                    |
| garden_plots    | Each plot in a user's garden: plant_id, growth_stage, last_watered       |
| plants          | Plant catalog: name, species, rarity, growth requirements                |
| user_plants     | Plants owned by user: plot assignment, dormancy status                   |
| garden_items    | Items owned by user: shards balance, Revival Potions, Fertilizer         |
| subscriptions   | Stripe subscription state per user                                       |

## WHAT NOT TO DO

- Do not build parent account linking.
- Do not build leaderboards.
- Do not build party/group study features.
- Do not build IB or SAT content.
- Do not build a diagnostic test.
- Do not call Anthropic API from client-side code.
- Do not switch Stripe to live mode.
- Do not add any UI library other than shadcn/ui and Tailwind.
- Do not add any charting library other than Recharts.
- Do not write raw SQL — use Supabase client methods with TypeScript types.
